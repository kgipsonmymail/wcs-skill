# Structure

## Top-Level Layout

- `SKILL.md`: canonical skill definition and execution contract
- `wcs-cn/`: Simplified Chinese 1:1 mirror of the skill
- `README.md`: public entry point and installation guidance
- `LICENSE`: open-source license
- `references/`: templates and operational references
- `assets/`: reserved for future repository assets
- `scripts/`: reserved for future helper scripts
- `docs/`: repository governance, status, and maintenance records

## Reference Files

- `references/api_reference.md`: index for loading WCS references
- `references/core_docs_template.md`: baseline doc-set template
- `references/coding_standard_template.md`: portable coding standard template
- `references/workflow_checklists.md`: end-to-end delivery checklists

## Maintainer Navigation

- Change the skill behavior in `SKILL.md`
- Keep `wcs-cn/` structurally and semantically aligned when the English skill changes
- Change reusable process guidance in `references/`
- Record repository-level changes in `docs/dev_log.md`
- Keep public positioning in `README.md`
